BlowTorch User Manual
=====================

Dot commands
------------
Lines that start with a single "." are handled by BlowTorch when "process period"
is enabled (default). Type ".." alone to toggle that processing on or off.
Prefix a server command with ".." to send a leading "." to the game without
running a client command (e.g. "..look" sends ".look").

Aliases that share a name with a command win when you type ".name newtext"
(that updates the alias). Plugins may register additional commands via
RegisterSpecialCommand; those appear only while the plugin is loaded.

Built-in commands
-----------------

.colordebug <0|1|2|3>
  Color debug mode for ANSI diagnostics.
  0 = normal colors
  1 = color on, show codes
  2 = color off, show codes
  3 = color off, hide codes

.closewindow
  Requests a dirty exit of the game window (closes the session UI without a
  clean disconnect sequence).

.timer <action> <name> [silent]
  Control a named timer. Actions: play, pause, reset, stop, info.
  Optional third token (any non-empty value) suppresses toast feedback
  (except info). Example: .timer play mytimer
  Example silent: .timer play mytimer silent

.dobell
  Triggers the configured bell reaction (same path as a received BEL).

.togglefullscreen
  Toggles the fullscreen preference.

.wrap [on|off]
  Controls whether the input bar grows with multiline text (default on).
  With no argument, prints current state and usage.
  Also: Options → Input → Grow Input Bar? (not Word Wrap?).

.gmcp …
  GMCP helpers. GMCP is an optional out-of-band telnet channel (option 201)
  for structured data (vitals, room, …). Servers differ.
  Enable: Options → Service → GMCP Options (Use GMCP?, Supports String, Log GMCP?).
  Subcommands:
    .gmcp                 help
    .gmcp status          current flags
    .gmcp sniff [on|off]  log handshake/packets to app error log
                          (files/logs/blowtorch2.log; also session log if on.
                          prints absolute path when enabled; Crash report → Show log)
    .gmcp sniff tail [N]  show last N GMCP lines from that log in-game (0-100, default 40)
    .gmcp version         client hello / syntax notes
    .gmcp supports […]   show or set supports modules
    .gmcp dump [path]     dump cached GMCP table
    .gmcp send <payload>  queue a GMCP packet
  Lua: Send_GMCP_Packet("module {…}")
  Triggers: pattern starting with % (default GMCP trigger character)

.keyboard …   (alias: .kb)
  Manipulate the input bar. With no arguments, prints usage help.
  Text: add, popup, flush, close, clear
  Edit: sel | selectall, cut, copy, paste
  Cursor: start | cursorstart, end | cursorend,
          stepf | stepr (right one char), stepb | stepl (left),
          stepu (up one line), stepd (down one line)
  Also: Edit button opens Sel/Cut/Copy/Paste and a ← ↑ ↓ → pad.
  Examples:
    .kb popup reply
    .kb add foo
    .kb flush
    .kb sel
    .kb cut
    .kb start
    .kb end
    .kb stepf
    .kb stepb
    .kb clear
    .kb close

.disconnect
  Shows a local "Disconnected." notice in the game window.
  Prefer the overflow menu Disconnect for a real disconnect, or use
  Reconnect/Disconnect there as needed.

.reconnect
  Shows a local "Reconnecting . . ." notice in the game window.
  Prefer the overflow menu Reconnect to reconnect the session.

.run <directions>
  Speedwalk. Direction letters map through Speedwalk Directions settings.
  Defaults: n e s w u d, h=nw, j=ne, k=sw, l=se. Prefix letters with a count.
  Insert free-text commands between commas.
  Examples:
    .run 3desw2n
    .run 3ds,open door,3w
  With no argument, prints usage and the default mapping.

.loadset <setname>
  Built-in stub; when the button_window plugin is loaded it registers this
  command and loads the named button set.

.clearbuttons
  Built-in clears all on-screen buttons. When button_window is loaded it
  re-registers this command and clears via the button window.

.switch <connection>
  Switch the foreground UI to another open connection by display name.

.search <phrase>
.search 'phrase with spaces'
.search "phrase"
.search next | n
.search prev | previous | p
.search close | hide | clear
  Opens or drives scrollback search. Empty argument opens the search UI.
  Buttons may also use: /search 'phrase'

On-screen buttons (button_window)
---------------------------------
Edit button layout: open ⋮ → **Edit buttons**, or long-press the ⋮ next to
Edit/Send. In edit mode, Cancel is on the left and Done on the right of the ⋮
strip; the gear opens set options (grid, gesture-hint badges, defaults).

Tap runs the primary command. Extra gestures (edit a button → Swipe / Accordion):

  Swipe up / down / left / right
    Each direction can run a different command (overrides classic Flip).
    Drag ~finger-width off the button to fire that direction.

  Hold
    Optional hold-command after a short press-and-hold.

  Accordion
    Up to 5 child buttons expand from a parent (direction + tap/hold/swipe
    trigger). Great for packing many macros into one tile. Badges in the
    editor: T = tap, H = hold, S = swipe. Options can show gesture hint
    arrows on-screen (uncheck to hide U/D/L/R and Hold markers).

Plugin commands (when loaded)
-----------------------------

button_window (default Free build):
  .loadset <name>       Load button set
  .clearbuttons         Clear buttons via button window

Menus (session overflow)
------------------------
Edit buttons — Enter on-screen button layout edit mode
Crash report — Show log / Share log (app error log)
About/donate — Version and description
Help — This manual

Connection duration appears on the ongoing connection notification and on the
launcher row (uptime while connected; last duration after disconnect).
Background keepalive: Wi-Fi lock + CPU wake lock; Options → Service →
Battery optimization… (or a one-shot prompt when connected).
Notification responders: bundled soft/mid/loud sounds + pick file from storage.

See also: docs/options-guide.md for Options groups (Service session log, GMCP,
Miscellaneous storage, battery, notification sounds, etc.).
